ORLO Store · Dubai

Elevation Proposal.

A focused roadmap to take the site from well-crafted small-biz e-commerce to a full Hermès / Aesop editorial register. Subtraction and discipline, not addition.

Version 01 · 2026 · prepared for Rami

Three principles, applied everywhere

1. Subtraction over addition. Every surface should have one focal moment, not three. Most existing pages have right palette + right type but slightly too much information density.

2. Discipline over variety. Same rules on every surface — photography on warm beige, italic serif for product names, sans for system text, one coral accent per page. Repetition is what makes a brand.

3. Performance is luxury. Instant page transitions, no layout shift, no font swap, no flicker — anything else breaks the spell and reads as small biz.

01UX · interactions

Small but persistent friction points the customer notices. Fixing these is the single highest-ROI bucket — minimal effort, perceptible quality jump.

Form validation — inline branded login / signup / forgot / reset / verify

Replace browser-native HTML5 tooltips (English-only system-UI boxes) with inline bilingual error messages under each field — DM Sans navy + Almarai grey, coral underline accent on the field.

Currently every auth page breaks the editorial register the instant the user mis-types. Five pages, ~40 lines each.

Next High impact Medium effort

Cart qty-limit message — fix silent failure cart.html

Cart calls showOrloToast (function doesn't exist) AND the page has no toast container — so hitting the stock cap or 10-per-order cap shows nothing. Fix: 5-line patch to use the same inline tooltip the PDP already uses.

Currently customers think the + button is broken. Easiest quality win on the site.

Now High impact Small effort

Loading states — extend the checkout pattern PDP add-to-cart, account actions

Cart checkout buttons now show "Verifying stock…" during the round-trip. Apply the same pattern to: PDP Add to Cart (when stock-checking), account-page Save / Cancel / Return submit buttons, address edit.

Anywhere a button triggers a server call > 500ms should signal "we're working" instead of looking frozen. Hermès does this on every async action.

Next Medium impact Medium effort

Pill curvature unification — Option 2 sitewide ~24 surfaces

Apply the "soft-rectangle" curvature (25% of element height) to every interactive rounded surface — same softness everywhere. Excludes cards / containers / variant tiles per earlier decision.

Single visual language across the entire site. Already mocked at /pill-mockups.html; awaiting your call on row-by-row approval.

Later Medium impact Medium effort

02Content · voice

The biggest invisible elevator. Right palette + right type can't save copy that reads like Amazon. Hermès names every product like a museum label — italic serif, one descriptive line, no marketing adjectives.

Product naming convention ~all products

Audit every product name. Replace marketing-flavoured names ("Classic Vintage Charm Bug") with editorial labels ("Vintage Beetle, Hippie Flowers"). Add a one-sentence Cormorant italic description ("A reminder that joy used to come painted in petals"). Drop bullet specs from card view.

Hermès doesn't say "stylish statement piece" — they describe what it is and let the photograph do the selling. Rename in batches of 10, audit visually before/after.

Next High impact Large effort

"The Edit" — populate with real content orlo-the-edit.html

The Edit page exists as a shell. Populate with 3–5 short editorial pieces (300–500 words each, Cormorant italic h1, DM Sans body): "Why we chose the camper van", "Vintage objects in a rented apartment", "On imperfection in handmade things", etc. Each links to relevant products inline.

This is what turns the site from "curated shop" into "editorial brand." Without it, ORLO is a marketplace with nicer photos. With it, ORLO is something customers come back to read.

Next High impact Large effort

Voice / copy guide brand-guide.html § new

Add a section to the brand guide documenting the editorial voice — sentence cadence, what to say / not say, product description structure, three tone-of-voice examples (catalogue card / PDP / video caption).

Without this, every new product name drifts back to marketing-speak. Same logic as the photography spec — locked rules prevent regression.

Later Medium impact Medium effort

03Visual · photography discipline

The brand guide locks the spec; execution is the gap. A site with 20 perfectly-shot products beats one with 200 inconsistent ones.

Photography audit all current products

Go through every product image. Flag any not on warm beige #E5DCCF, any with props that leaked in, any wrong aspect ratio. Re-generate via Gemini with the v04 catalogue prompt for any that don't match. Tag each product as "approved" or "needs reshoot."

The visual register lives or dies on this. One off-brand product image in the grid breaks the whole register for the viewer.

Next High impact Large effort

Generate the 3-image set for every product brand-guide §03

Per the brand guide: 1 clean catalogue + 1 dimension card + 1 lifestyle (4:5 + 9:16 same scene). Use the Spec Prompt admin tool for dimensions, lifestyle prompt for editorial.

Currently most products have only the clean shot. PDP feels thin without the dimensions and lifestyle. Three images per product is the minimum complete set.

Later Medium impact Large effort

Hero — two-size upload in admin heroes admin + index.html

Add a second upload slot to Heroes admin for the mobile 4:5 variant. Update index.html hero markup to use <picture> with media query so the right one serves per viewport. Eliminates the current cropping loss.

Already discussed and approved in concept. ~30 min of work. Means future hero activations don't require touching index.html ever again.

Later Medium impact Small effort

04Performance · Core Web Vitals

Premium feels instant. Layout shift, font swap, image popping — all of these read as small biz no matter how good the design is.

Audit Core Web Vitals against luxury benchmarks PageSpeed Insights

Run PageSpeed Insights on the homepage, PDP, and cart. Target: LCP < 1.5s, CLS < 0.05, INP < 200ms. Anything over is a Hermès gap.

Hermès loads instantly with zero shift. Without this audit, we don't know how close we are.

Next High impact Small effort

Bundle analysis — find more dead code app.js, app.min.js, product.js

Continue the cleanup started this sprint. Candidate: the entire legacy renderProducts() function in app.js (~80–100 lines, overridden by v2 on the only page that uses it). Audit which other functions are dead on the live site.

Less JS to parse = faster TTI. Every kilobyte matters on UAE 4G.

Later Medium impact Medium effort

Image lazy-loading + sizes verification all surfaces

Verify every image below the fold has loading="lazy". Verify the LCP image has fetchpriority="high" (we already do this for hero). Verify srcset + sizes are set so phones don't download desktop-sized images.

Easy wins. Each ~5 minutes per surface to audit.

Later Medium impact Small effort

05Post-purchase · the forgotten 30%

Every brand polishes the homepage. The brands that feel premium ALSO polish the receipt email, the shipping notification, the cancel page — the moments customers actually read carefully.

Transactional email templates order confirm / shipping / cancel

Current state: probably default Stripe receipts. Build branded HTML email templates — cream background, navy + coral, Cormorant italic order title, DM Sans line items, bilingual. Three templates: order placed, order shipped, order cancelled.

Customers re-read order emails several times. Stripe-default look undoes the website's good work in 1 second.

Next High impact Large effort

Success page after checkout success.html

Audit current state and align to editorial register. Cormorant italic confirmation headline, soft thank-you in Arabic + English, order number in DM Sans 500, single "Continue browsing" CTA. No celebrations, no confetti, no marketing upsell.

Later Medium impact Small effort

Account page polish account.html

Account is the second-most-visited page after homepage for returning customers. Audit modal styles, button radii (per the pill plan), spacing, address card layout. Make sure the editorial register holds when the customer is looking at their own data, not products.
