Email templates · before / after

Same parameter API, same copy, same triggers — only the visual register changes from the v1 marketing look (dark navy gradient header, coral pill, Inter font, big emoji) to v2 editorial (cream surface, Cormorant Garamond italic, thin coral hairline, navy minimal pill, no shadows or rounded corners).

Stripe receipts are NOT in this preview. Stripe sends its own receipt + invoice PDFs directly (the dark-navy "Receipt from ORLO Store" email with PDF attachments) — those live in the Stripe Dashboard and can only be customised there (logo + brand color + footer text). They're separate from the templates here.

The function signatures stay identical — customerEmail({origin, icon, titleEn, bodyEn, bodyAr, infoBoxEn, infoBoxAr, ctaUrl, ctaText, ...}) — so all 11 callers (webhook.js, ship-order.js, return flows, auth flows, etc.) work without changes. The legacy parameters ctaColor, infoBoxBg, infoBoxBorder are accepted for back-compat but quietly ignored in v2 so the editorial look stays consistent across every email.
