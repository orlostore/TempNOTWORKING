Proposal — three message channels, each with one rule:
  • Inline-under-field (form validation) — login / signup / forgot-password / reset-password / verify-email. Replaces every native browser tooltip with branded bilingual text under the offending field, coral underline accent on the input. ~40 lines per form.
  • Inline-floating-tooltip (stock / limit messages near a qty control) — PDP qty stepper (already works) AND cart qty stepper (currently broken). Fix cart to use the same pattern: 5-line patch to cart.html's mutateQty() — replace the dead showOrloToast call with a small DOM-injected tooltip identical to the PDP one.
  • Top-right toast (orloToast) — system events: order cancelled, return submitted, address error, payment failed, etc. Already works on pages that include the container. Keep as-is.

No mockup file modifies live code. When you approve, the work is: (a) build the inline-form-validation system on login/signup/forgot/reset/verify, (b) fix the cart qty-limit tooltip. Each is an independent commit.
